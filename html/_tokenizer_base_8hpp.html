TokenizerBase.hpp File Reference

Contains the TokenizerBase class; a generic virtual tokenizer.
## Description #
The tokenization process is split into 2 distinct phases:
- Scanner Stage char -> LEXEME
- Actual Tokenization LEXEME -> TokenType
The TokenizerBase class should be specialized by inheritence.
Derived objects should override the virtual getNext function that handles the Actual Tokenization LEXEME -> TokenType phase.
Other functions may also be overridden, but most file formats shouldn't require it.
See the TokenizerINI class for an example.

## Standards & Requirements #
- All token containers must have a token with TokenType::END at the end, to represent EOF.
This is handled by the tokenize function, which appends a blank token with TokenType::END.
- The virtual getNext function must be self-contained, and not require the end user to call other functions.
Excluding the hasMore function,
- The virtual getNext function may use recursion.
- The virtual getNext function must return a fully-constructed Token object, and contain the string used to detect it.
- It is recommended that the tokenization process does not verify whether a file is correctly formatted, only that tokens are correctly formatted: For example, quotation marks & brackets are closed properly. The format verification process should occur during the parsing process.
- Derived objects should be members of the token namespace, or a sub-namespace. More...
