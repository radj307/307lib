8  * # TokenRedux
9  *
10  * This is a universal tokenization & parsing library that uses templated objects and stringstreams to parse data into
11  * any implementation-defined object type.
12  *
13  * ## Packages
14  *
15  * Packages are collections of objects that perform a similar, specific function.
16  *
17  * ### Framework Package
18  *
19  * The base virtual objects that should be inherited from to implement the tokenizer/parser pipeline.
20  * This also provides all of the base functions available to the Tokenizer and the Parser.
21  * In almost all cases, the framework provided by <TokenRedux.hpp> is adequate.
22  *
23  * ### Definition Packages
24  *
25  * Provides lexeme and token types that can be reused between any number of file formats.
26  *
27  * #### Types
28  *
29  * ##### Objects
30  * - __Lexeme Dictionary__
31  * Provides bindings between characters and their associated Lexeme types that are usable by the Tokenizer.
32  * Must be derived from, or implement the signature of, the `LexemeDictBase` object.
33  * - __Token__
34  * Container object that holds a character sequence and a single type provided by the __Token Type__ enum.
35  * These are read in a list by the parser, which looks for specific sequences of tokens to both validate and parse data.
36  * Must be derived from, or implement the signature of, the `TokenBase` object.
37  *
38  * ##### Enums
39  * - __Lexeme Type__
40  * Defines possible types associated with a single character, such as a digit, quotation mark, or letter.
41  * - __Token Type__
42  * Defines possible types associated with sections of the input data, such as words, numbers, or lines.
43  *
44  * ### Format Packages
45  *
46  * These provide the actual tokenizer and parser implementations used for a single file format.
47  * Format Packages are dependent on a single _Definition Package_, and cannot be transferred without modifying code.
48  * This acts like an interface for retrieving usable data from tokenized data.
49  *
50  * #### Types
51  *
52  * ##### Objects
53  * - __Tokenizer__
54  * Takes a stringstream as input, and divides it into a vector of Tokens using the provided Definition Package.
55  * Must derive from the `TokenizerBase` object.
56  * - __TokenParser__
57  * Takes a vector of Tokens as input, and parses it into a usable object type, defined by the implementation.
58  * Must derive from the `TokenParserBase` object.
59  *
60  * # Getting Started
61  *
62  * ## Planning
63  *

65  *
66  * ## Building a Format Package
67  *
68  * 1. Start by creating a new `Tokenizer`-derived type, filling in template arguments with the corresponding
69  object provided by the _Definitions Package_ you're using, and override the `getNext()` function.
70  * ```cpp
71  * class Tokenizer : token::base::TokenizerBase<Lexeme, LexemeDictionary, Token, Token> {

81  * - Use the TokenRedux parser base object.
82  * 1. Choose the object/type you want the parser to return. This is usually a list of some sort, or a custom object.
83  * 2.
