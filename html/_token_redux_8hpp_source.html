1 /**
2  * @file TokenRedux.hpp
3  * @author radj307
4  * @brief New tokenization & parsing library providing superior abstraction capabilities and a much less confusing usage experience.
5  * For more information on tokenization, see: https://en.wikipedia.org/wiki/Lexical_analysis#Tokenization
6  *
7  * ## Terminology
8  *
9  * | Term | Description |
10  * |---------|-----------------------------------------------------------------------------------------------------|
11  * | Format | A data format, usually a file format, such as JSON, YAML, INI, etc. |
12  * | Package | Contains the actual implementations of some portion of the TokenRedux pipeline. |
13  * | Lexeme | A single character's type, such as a lowercase letter, digit, or semicolon. |
14  * | TokenBase | Composed of 0 or more characters with a similar lexical type, such as a string, boolean, or number. |
15  *
16  *
17  * ## TokenRedux Components
18  *
19  * | Component / Layer | Functionality |
20  * |--------------------|----------------------------------------------------------------------------------------|
21  * | Virtualized Base | Provides the basic tokenization framework, as well as helper methods. |
22  * | Definition Package | Provides generic Lexeme definitions and TokenBase types that apply to one or more formats. |
23  * | Format Package | Provides the Tokenization and parsing implementations specific to one format. |
24  *
25  *
26  * ## Definition Packages
27  *
28  * A definitions package is usually composed of a LexemeDictBase override, and 2 enums:
29  * - __Lexeme Type__
30  * Defines possible types associated with a single character, such as a digit, quotation mark, or letter.
31  * - __Token Type__
32  * Defines possible types associated with sections of the input data, such as words, numbers, or lines.
33  *
34  *
35  * ## Format Packages
36  *
37  * A format package is composed of 2 derived objects, to implement the tokenization & parsing stages of reading a single serialized data format.
38  * - __Tokenizer__
39  * Takes a stringstream as input, and divides it into a vector of Tokens using the provided Definition Package.
40  * - __Parser__
41  * Takes a vector of Tokens as input, and parses it into a usable type, defined by the implementation.
42  */
