7 # How to use Version 3
8 You can still use TermAPI v3 by setting `TERMAPI_VERSION_3` to `ON` before adding 307lib as a subdirectory in CMake.
9 Example:
10 ```cmake
11 set(TERMAPI_VERSION 3)
12 add_subdirectory("307lib")
13 ```

15 # Changes in Version 4
16 - Removed `str-lib` dependency.
17 - Renamed the `sys::term` namespace to `term`
18 - Major overhaul of library structure. Headers have been split into the following object libraries, which are linked to the TermAPI library.
19  - __ANSI__
20  Contains some helper objects & functions for working with ANSI escape sequences.
21  - __color__
22  Contains the color library.
23  - __term__
24  Contains the `sys::term` namespace from v3, now renamed to just `term`.
25  Much of the problematic modularity from v3 is now gone.
26  - __extensions__
27  Extra headers that require libraries other than `shared`.
28  To enable extensions, `ENABLE_FILELIB` must be `ON`, then set `TERMAPI_ENABLE_EXTENSIONS` to `ON` in CMakeLists.txt before you add the "307lib" subdirectory.
29 - Object libraries are automatically linked to the `TermAPI` library, don't include the object libraries directly unless you need specific features from them.
30 - Replaced the `Sequence` object with a using statement.
31 - Added wstring support to most/all functions with the `wSequence` type.
32 - Sequences can no longer be called on their own, you're responsible for inserting sequence strings into STDOUT.
33 - Optimizations & improvements across the board.
34 - Added more color definitions in `color-values.h`
