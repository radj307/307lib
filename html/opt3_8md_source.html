3 # opt3
4 
5 The `opt3` namespace provides everything you need to manage and utilize GNU-style commandline arguments independently of platform or compiler.
6 It is intended to replace the deprecated `optlib` library, and consists of a single header file (`opt3.hpp`).
7 Migrating an existing project from the now-deprecated optlib will likely require some code changes.
8 
9 ## Basic Concepts
10 
11 ### Argument Types
12 
13 One of three *argument types* is assigned to each parsed argument, which corresponds to its semantics and how it is interpreted.
14 
15 - Option
16  - Always preceded by two delimiters `--`.
17  - Can be of any length.
18  - Is able to capture parameters.
19 - Flag
20  - Always preceded by one delimiter `-`.
21  - Always a single character in length.
22  - Multiple flags can be chained together.
23  *Ex: `-qv`* is equivalent to *`-q -v`*
24  - Is able to capture parameters as long as it is not part of a chain, or is the last element in the chain.
25 - Parameter
26  - Any argument that is not an option or a flag.
27  - Note that negative numbers *(ex: `-20`)* may result in unexpected behaviour because they may be interpreted as Flags.
28 
29 ### Capturing Arguments
30 
31 Option and Flag arguments can be made to *capture* Parameters. Captured parameters are not included directly in the list of arguments; instead they are stored within the argument that captured them.
32 Capturing arguments must be specified in the constructor of `opt3::ArgManager`, but they can be specified in a number of ways.
33 
34 #### Defining Capturing Arguments
35 All capturing arguments must be specified in the constructor of `opt3::ArgManager`.
36 If a Flag/Option is *not* specified, it will never be able to capture under any circumstances.
37 
38 The simplest way is to pass all of the Flags and Options in your program that *can* capture to the `opt3::ArgManager` constructor:
39 ```cpp
40 int main(const int argc, char** argv)
41 {
42  opt3::ArgManager args{ argc, argv, 'h', "help" };
43 }
44 ```
45 You can force a specific Flag or Option to *require* capture input by using custom string literals:
46 ```cpp
47 int main(const int argc, char** argv)
48 {
49  using namespace opt3_literals;
50  opt3::ArgManager args{ argc, argv, 'h'_requireCapture, "help"_reqcap };
51  // ""_requireCapture & ""_reqcap can be used interchangeably.
52 }
53 ```
54 Other custom string literals include:
55 - `""_requireCapture` / `""_reqcap`
56 - `""_optionalCapture` / `""_optcap`
57 - `""_noCapture` / `""_nocap`
58 
59 Alternatively, you can use *template arguments* to define a set of arguments that have similar rules.
60 ```cpp
61 int main(const int argc, char** argv)
62 {
63  opt3::ArgManager args{ argc, argv,
64  opt3::make_template(opt3::CaptureStyle::Optional, opt3::ConflictStyle::, 'h', "help")
65  };
66 }
67 ```
68 
69 #### *Explicit* vs. *Implicit* Capture
70 
71 A capture can be classified as either explicit or implicit. An explicit capture involves appending an equals sign `=` followed by a string. On the other hand, an implicit capture captures a Parameter that is separated by at least one space character.
72 - Explicit: `--arg="Hello World!"`
73 - Implicit: `--arg "Hello World!"`
74 
75 #### Capture Style
76 The `CaptureStyle` bitfield enum defines capturing rules for a specific argument or templated argument.
77 
78 - Disabled
79  - Disables implicit and explicit captures for this argument/template.
80  - If a user attempts to force an argument to capture by using an explicit capture, the value that would have been captured is treated like a Parameter and is added directly to the argument list.
81 - EqualsOnly
82  - Disables implicit captures for this argument/template.
83 - Optional *(Default)*
84 - Required
85  - An exception is thrown if this argument does not specify a capture value.
86 
87 ### Argument Parsing Rules
88 
89 You can use the `ArgumentParsingRules` structure to get more control over the argument parsing process.
90 It is highly recommended that you do not call any of this object's methods yourself; they are intended for internal use only.
91 
92 The following members are available:
93 
94 - `delimiters`
95  - Defines the list of characters (`char`) that are considered valid Flag/Option prefixes.
96  - Default: { '-' }
97 - `allowNumericFlags`
98  - Determines whether digits are allowed to be considered flags. This fixes a potential bug where Parameters are interpreted as Flags when entering negative numbers followed by alphanumeric characters in the same string, ex: `-200.0F`.
99  - Does nothing if `-` is not a delimiter.
100  - Default: false
101 - `assumeValidNumberWithDashPrefixIsNegative`
102  - A valid negative number *(ex: `-30`)* would normally be treated like a Flag and interpreted as `-3 -0`. When this is true, valid negative numbers are interpreted as Parameters instead of flags.
103  - Does nothing if `-` is not a delimiter.
104  - Default: true
