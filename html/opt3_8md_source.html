3 # opt3
4 
5 The `opt3` namespace provides everything you need to manage and utilize GNU-style commandline arguments independently of platform or compiler.
6 
7 opt3 makes extensive use of modern C++ features like [concepts](https://en.cppreference.com/w/cpp/language/constraints), [variadic functions & templates](https://en.cppreference.com/w/cpp/utility/variadic), [variants](https://en.cppreference.com/w/cpp/utility/variant), and [optionals](https://en.cppreference.com/w/cpp/utility/optional).
8 It is not the most efficient library out there by any means, but it makes up for it by being easy to set up & use in a wide variety of applications.
9 
10 ## Getting Started
11 
12 The opt3 library revolves around the `ArgManager` *(`arg_manager`)* class. `ArgManager` inherits from the `arg_container` class, which provides a comprehensive set of functions specifically designed to handle command-line arguments effectively. Documentation on these functions can be found [here](https://radj307.github.io/307lib/html/structopt3_1_1arg__container.html).
13 Creating a new `ArgManager` instance is simple, and only *requires* the argument count (`argc`) and argument array (`argv`) from your program's `main()` function.
14 ```cpp
15 #include <opt3.hpp>
16 
17 int main(const int argc, char** argv)
18 {
19  opt3::ArgManager args{ argc, argv }; //< ready to use!
20 }
21 ```

23 ## Basic Concepts
24 
25 See [here](https://www.learncpp.com/cpp-tutorial/command-line-arguments/) for an introduction to commandline arguments in C++.
26 
27 The ArgManager class constructor parses the `argc` and `argv` parameters from your `main()` function into a vector of `std::variant` type, preserving argument precedence. Each `std::variant` can represent an `opt3::Option`, `opt3::Flag`, or `opt3::Parameter` (refer to [Argument Types](#argument-types)). The entire parsing process happens during the instantiation of `ArgManager`.
28 
29 Options and Flags can capture additional input, which is stored as an optional member within the corresponding `opt3::Option` or `opt3::Flag` object. Captured input is denoted by appending an equals sign (`=`) to the option/flag name. For example, `--opt=someCapturedString` is an `opt3::Option` named `opt` that captures the string `someCapturedString`. This style of capturing is called *explicit* capturing.
30 Alternatively, Options/Flags can be configured to capture Parameters that immediately follow them by providing the option/flag name to the constructor of `ArgManager`. This style of capturing is called *implicit* capturing. If a parameter is captured by an option/flag, it is removed from the direct argument list. Therefore, in the example `--opt someCapturedParam`, the `opt3::Option` named `opt` captures an `opt3::Parameter` named `someCapturedParam`, which is no longer present directly in the argument list. Instead, it can be accessed through the `opt` option. See [Defining Capturing Arguments](#defining-capturing-arguments) for more information.
31 
32 When referencing an argument in your code, only the argument's *name* is used. An argument's name does not include prefix characters like dashes (`-`). For instance, to refer to the flag "-h" you would use the char 'h' or string "h", **not** "-h". This allows your code to work independently of whatever prefix delimiters are being used.
33 
34 ### Argument Types
35 
36 One of three *argument types* is assigned to each parsed argument, which corresponds to its semantics and how it is interpreted.
37 
38 - Option
39  - Name is preceded by two delimiters `--`.
40  - Names can be any length greater than 0.
41  - Is able to capture parameters.
42 - Flag
43  - Name is preceded by one delimiter `-`.
44  - Names are always exactly 1 character in length.
45  - Is able to capture parameters.
46  - Multiple flags can be chained together. Only the last flag in the chain can capture parameters.
47  *Ex: `-qv`* is equivalent to *`-q -v`*
48 - Parameter
49  - Any argument that is not an option or a flag.
50  - Is able to be captured by Options/Flags. When a Parameter is captured, it is removed from the argument list.
51  - Note that negative numbers *(ex: `-20`)* may result in unexpected behaviour because they may be interpreted as Flags when the `assumeValidNumberWithDashPrefixIsNegative` parsing rule is set to `false` (see [Argument Parsing Rules](#argument-parsing-rules)).
52 
53 In the code, these arguments are templated instances of [`basic_arg_t`](https://radj307.github.io/307lib/html/structopt3_1_1basic__arg__t.html)
54 
55 #### Defining Capturing Arguments
56 
57 All capturing arguments must be specified in the constructor of `opt3::ArgManager`.
58 If a Flag/Option is *not* specified, it will never be able to capture under any circumstances.
59 
60 The simplest way is to pass all of the Flags and Options in your program that *can* capture to the `opt3::ArgManager` constructor:
61 ```cpp
62 int main(const int argc, char** argv)
63 {
64  opt3::ArgManager args{ argc, argv, 'h', "help" };
65 }
66 ```
67 You can force a specific Flag or Option to *require* capture input by using custom string literals:
68 ```cpp
69 int main(const int argc, char** argv)
70 {
71  using namespace opt3_literals;
72  opt3::ArgManager args{ argc, argv, 'h'_requireCapture, "help"_reqcap };
73  // ""_requireCapture & ""_reqcap can be used interchangeably.
74 }
75 ```
76 Other custom string literals include:
77 - `""_requireCapture` / `""_reqcap`
78 - `""_optionalCapture` / `""_optcap`
79 - `""_noCapture` / `""_nocap`
80 
81 Alternatively, you can use *template arguments* to define a set of arguments that have similar rules.
82 ```cpp
83 int main(const int argc, char** argv)
84 {
85  opt3::ArgManager args{ argc, argv,
86  opt3::make_template(opt3::CaptureStyle::Optional, opt3::ConflictStyle::, 'h', "help")
87  };
88 }
89 ```

91 #### *Explicit* vs. *Implicit* Capture
92 
93 A capture argument can be specified explicitly or implicitly. An explicit capture involves appending an equals sign `=` followed by a string. On the other hand, an implicit capture captures a Parameter that is separated by at least one space character.
94 - Explicit: `--arg="Hello World!"`
95 - Implicit: `--arg "Hello World!"`
96 
97 #### Capture Style
98 
99 The `CaptureStyle` bitfield enum defines capturing rules for a specific argument or templated argument.
100 
101 - Disabled
102  - Disables implicit and explicit captures for this argument/template.
103  - If a user attempts to force an argument to capture by using an explicit capture, the value that would have been captured is treated like a Parameter and is added directly to the argument list.
104 - EqualsOnly
105  - Disables implicit captures for this argument/template.
106 - Optional *(Default)*
107 - Required
108  - An exception is thrown if this argument does not specify a capture value.

110 ### Argument Parsing Rules
111 
112 If you want to change the way that opt3 parses commandline input, you can use the `ArgParsingRules` struct.
113 The `ArgManager` constructor accepts an `ArgParsingRules` instance as the third parameter.
114 ```cpp
115 int main(const int argc, char** argv)
116 {
117  opt3::ArgParsingRules rules;
118  rules.allowUnexpectedArgs = false;
119 
120  opt3::ArgManager args{ argc, argv, rules,
121  'h',
122  "help",
123  };
124 }
125 ```
126 
127 The following settings are exposed via `ArgParsingRules`:
128 
129 - `delimiters`
130  - Defines the list of characters that are considered valid Flag/Option prefixes.
131  - Default: `{ '-' }`
132 - `allowNumericFlags`
133  - Determines whether digits are allowed to be considered flags. This fixes a potential bug where Parameters are interpreted as Flags when entering negative numbers followed by alphanumeric characters in the same string, ex: `-200.0F`.
134  - This is ignored when `-` is not a delimiter.
135  - Default: `false`
136 - `assumeValidNumberWithDashPrefixIsNegative`
137  - A valid negative number *(ex: `-30`)* would normally be treated like a Flag and interpreted as `-3 -0`. When this is true, valid negative numbers are interpreted as Parameters instead of flags.
138  - This is ignored when `-` is not a delimiter.
139  - Default: `true`
140 - `defaultCaptureStyle`
141  - Determines the default capture style for arguments/templates in the capture list that did not have a capture style explicitly defined.
142  - Default: `Optional`
143 - `allowUnexpectedArgs`
144  - Determines what happens when the user enters an Option or Flag that wasn't explicitly defined in the capture list.
145  - When `true`, syntactically correct Options & Flags are always allowed regardless of whether or not they were defined in the capture list. When `false`, only Options & Flags that were explicitly defined in the capture list are allowed.
146  - The action taken for disallowed arguments depends on the value of `convertUnexpectedArgsToParameters`.
147  - Default: `true`
148 - `convertUnexpectedArgsToParameters`
149  - Determines what happens when unexpected arguments aren't allowed and an unexpected argument was entered by the user.
150  - When `true`, unexpected Options & Flags are converted to Parameters instead of triggering an exception. When `false`, unexpected arguments will cause an exception to be thrown by the parser.
151  - Disallowed arguments that were converted to Parameters will include prefix delimiters.
152  For example, in a project that expects `-h` but does **not** expect `-F`, the argument `-hF` would be interpreted as a Parameter with the value `-hF`.
153  - This is ignored when `allowUnexpectedArgs` is `true`.
154  - Default: `false`

158 ### Check if an argument was specified
159 
160 You can check if an argument was specified on the commandline by using the [`check`](https://radj307.github.io/307lib/html/structopt3_1_1arg__container.html#a686d6571318aae77d59524aef42e218e), [`check_any`](https://radj307.github.io/307lib/html/structopt3_1_1arg__container.html#ac7060604d1a29465ce34723f5dcf4216), & [`check_all`](https://radj307.github.io/307lib/html/structopt3_1_1arg__container.html#a52716edbccb679dca5149f60930e9b51) methods.
161 ```cpp
162 opt3::ArgManager args{ argc, argv };
163 
164 if (args.check('v')) {
165  // parameter "v" or flag "-v" was specified
166 }
167 if (args.check<opt3::Option>("help")) {
168  // option "--help" was specified
169 }
170 if (args.check_any<opt3::Flag, opt3::Option>('h', "help")) {
171  // flag "-h" or option "--help" was specified
172 }
173 if (args.check_all<opt3::Flag, opt3::Option>('h', "help")) {
174  // flag "-h" and option "--help" were specified
175 }
176 ```
177 
178 ### Check if an argument has a capture
179 
180 You can check if an argument has captured input by using the [`checkv`](https://radj307.github.io/307lib/html/structopt3_1_1arg__container.html#a48eadc0234a6c37abe5156b42632f100), [`checkv_any`](https://radj307.github.io/307lib/html/structopt3_1_1arg__container.html#a8e018c8dabeef85544cdacb688ed9d94), & [`checkv_all`](https://radj307.github.io/307lib/html/structopt3_1_1arg__container.html#ad4ae2185b17551ab032074953786bb0c) methods.
181 ```cpp
182 opt3::ArgManager args{ argc, argv,
183  'f',
184  "opt"
185 };
186 
187 if (args.checkv('f')) {
188  // flag "-f" has a capture argument
189 }
190 if (args.checkv_any('f', "opt")) {
191  // flag "-f" or option "--opt" has a capture argument
192 }
193 if (args.checkv_all('f', "opt")) {
194  // flag "-f" and option "--opt" have a capture argument
195 }
196 ```
