3 # 307Lib
4 This is my personal collection of libraries for C++.
5 
6 These were all converted to build with CMake using the "Visual Studio 17 2022" generator, and changing CMake options is not yet guaranteed to work without issue.

21 ## Shared Headers
22 These are some of the shared lib's headers:
23 
24 - sysarch.h
25  Used as a basic wrapper for certain preprocessor macros that can determine which C/C++ version the compiler is using, which compiler is being used, which operating system the compiler is running on, which architecture the compiler is set to, and more.
26 
27 - var.hpp
28  Contains some generic functions & concepts for working with variadic functions and tuples.
29 
30 - iter.hpp
31  Contains some generic functions related to iterators.
32 
33 - math.hpp
34  Contains generic math & algorithm functions.
35 
36 - make_exception.hpp
37  This is a wrapper based on Microsoft's non-standard implementation of std::exception to allow throwing generic exceptions with a message.
38  It accepts any number of arguments so long as they can be inserted into a std::ostream, and returns an _except_ struct, which is a child of std::exception.
39  To catch exceptions thrown with this method, any `catch(const std::exception& ex)` block will work.
40 
41 - Windows.hpp
42  Somewhat obviously, this is only relevant for Windows.
43  Experimental wrapper for the <Windows.h> header that includes it within a namespace to avoid polluting global.
44  Unfortunately, since most/all other libraries just include <Windows.h> in the global namespace this often will not work correctly.
45  It also undefines many of the <Windows.h> macros.
46 
47 - xRand.hpp
48  Provides a wrapper for <random> library functions.
49 
50 - process.hpp
51  Contains the process namespace & Proc object, which can be used to execute shell commands.
